Especificaciones Técnicas: Sistema de Landing Pages Dinámicas
1. Stack Tecnológico

Frontend: HTML5, CSS3 con Tailwind CSS (vía CDN para optimización de recursos).

Hosting: Despliegue estático en Netlify o GitHub Pages para garantizar alta disponibilidad, latencia mínima y certificados SSL gratuitos.

Integración de Comunicación: Protocolo WhatsApp Click to Chat ([https://wa.me/](https://wa.me/)[número]) para conversión inmediata.

2. Arquitectura de la Landing Page
La página debe seguir una estructura de sección única (Single Page) optimizada para dispositivos móviles:

Hero Section: Imagen principal del inmueble (formato object-cover para consistencia visual).

Header Data: Título descriptivo, precio formateado y tabla de especificaciones técnicas (Habitaciones, Baños, Metros cuadrados).

Cuerpo: Descripción técnica del inmueble con énfasis en características de venta.

CTA (Call to Action): Botón flotante o bloque inferior con ícono de WhatsApp; debe contener un mensaje preconfigurado (ej: text=Hola, vi la casa en venta...) para trazabilidad del lead.

3. Flujo de Operación del Sistema

Generación de Activos: El repositorio debe permitir la creación de páginas nuevas mediante la duplicación de la plantilla base y la actualización de variables en el DOM.

Vinculación Física: Generación de códigos QR mediante servicio de API externa, enlazando la URL de despliegue específica de cada inmueble.

Optimización de Recursos: El código debe evitar el uso de frameworks pesados (React/Angular) para asegurar tiempos de carga menores a 2 segundos en redes móviles 3G/4G.

4. Entregables del Equipo

Plantilla Maestra: Archivo index.html estandarizado con placeholders para src de imágenes y parámetros de contacto.

Documentación de Despliegue: Guía rápida de comandos para git push o arrastre manual en panel de Netlify.

Gestor de Assets: Estructura de directorios para imágenes optimizadas (compresión de peso para mejorar el Load Time)
